Redesign of Eco Shop Website

ABOUT THE PROJECT

The project has been started in a team of three. Our request was to make a presentation of work with client based on proposed changes on their website. We chose the website by ourselves having website topic and sham mail from our customer. The main purpose of redesign was to make increase conversion. As convesion can be understood in different ways we asked what does it mean for our "client" - as we expected it meant to make more clients make shopping on page. The biggest competitor for our "client" was https://minimal.com.pl that sells 10-12% more than them. Other pages that "client" gave us as his inspiration are:
https://www.rajapack.co.uk/paper_vs_plastic.html
https://www.nytimes.com/paidpost/allbirds/the-view-from-above.html
https://drinkeverandever.com
Our aim was mostly around young people interested in ecology. So we knew the purpose of redesign and shops that owner sees as inspiration and competition. Durign project we made tests with users, benchmark, owner site audit and then started to work on changes. The project by it's own didn't expect from as full redesign of website - just propositions of changes and possible sketches that will be used during 30 minutes presentation of our work. I've decided to go a little further and made a prototype expanding work that have been made during project itself. It's still not a full redesign of website - I made it to show my skills and thinking process expanding my portfolio.

CHANGES AND REASONS

Main Menu


Information Architecture
We've redesigned Main Menu, because as you can see on picture above the one that is on the page is more than 2 times longer than computer screen size.
During tests with users we asked them about the difference they sees between: "super food", "healthy food" and "(company name) delicatessen" - the answers to this question didn't reflect content of these subcategories.
Based on benchmark through propositions given by "Client" and some research on the internet, knowing that they have many categories as team we decided to cut categories that can be put together and from all of possibilities we made 3 main categories:
- "Delikatesy" (Delicatessen) - is all about food.
- "Drogeria" (Drugstore) - is all about cosmetics.
- "Dietetyk Online" (Online Dietitian) - it didn't fit to any above categories.
Second thing that I've made is adeed main menu if the page is scrolled up (it isn't working perfectly in prototype, but it shows the idea). This menu navigation bar is shown without advertisement because then it would take a lot of screen space - it would be annoying for users. The redesign of Main Menu has been shown in Prototype After Redesign Link

Contrast:

Text on website seemed to be really bright while appearing on white background so it was really hard to read something longer than few words. So we've made contrast tests for different colors appearing on website (Explanation: WCAG 2.0 level AA requires a contrast ratio of at least 4.5:1 for normal text and 3:1 for large text. WCAG 2.1 requires a contrast ratio of at least 3:1 for graphics and user interface components (such as form input borders). WCAG Level AAA requires a contrast ratio of at least 7:1 for normal text and 4.5:1 for large text.):

#D3B896
This color has been used for headers of different categories above products. As you can see in "Before Redesign Link" they are hard to notice while going through page.


#A96842
This color is used to write most of the text on the website. As you can see it doesn't pass the test for longer texts and font size below 18pt.
#FCO165
This color is used only for highlighting text or hovering feedback on clickable elements. It passes the first level test so we decided to leave it as it is.
#713F00
This color passes all of the tests and appears as part of the website. We decided to swipe the color of headers for this so all #D3B896 elements were changed for #713F00.
#70430B
This color appears as the one with the best contrast level above all on the website, it passes all the tests - so we decided to make it a color for all texts that appeard beyond headers as we used #713F00 over there. So the change is #A96842 to #70430B.
With font color changes I've made changes on other elements to make then fit to the rest of the page and make them more visible.

Categories



As you can see on the left old categories were always filled with all of possibilites. If user clicked one of categories we suppose that he want's to find something inside of it - so we show him what kinds of subcategories he or she can choose inside of one that he've been already interested about.

The other reason for such a choice is that it have to be above filters - it tells user where he is right now and because of longer list of categories if user wants to use filter he will have to scroll longer.

Basket Pop-up



If user puts something in the basket the basket window shows up. As you can see above even though I put 5 items in basket there is still just 1 item shown in pop-uping window. That could make some of users confused and discouraged - because the browser remembers items that have been put there even month or more before. Users could simply forget about that and later on they can feel being cheated and never come back to the website. Under item that is inside of the basket you can see advertisements of other items from the shop - as this window is made for users to see what did by clicking "Add to basket" button they can't really focus or even see it clearly.
During tests one of the tasks was to add 3 different items to the basket. The button that allows user to come back from the pop-up basket window is nearly being fully transparent. Users even though they don't want to stop shopping at this moment were pushed to do it and had to focus and think to find the button on the left that they searched for.
Third thing that I saw was hard to change is amount of items added to the list and possility to delete something that user wants to change for different item.
My proposition for this pop-uping window is to make clear and easy for user to see what he have already added to his basket. So I've deleted advertisements and I've added possibility to scroll through whole basket without searching for right button to see basket page. I've changed the color of button allowing users to come back to the previos viewed page (making it easier for them to continue shopping). As the last thing I've added possibility to change amount of items by plus and minus buttons.

Product Page



If someone came that far, he probably went this way: User comes to the page, searches for product, clicks on the product from the list - that person must be interested in this item. So our request is to make the clearest way to buy that thing as we can only imagine, without not required distractors.
During tests one of the users Didn't know for what reasone he have categories on the left - he felt this page overcrowded and annoying.
My proposition is to take off elements that can distract, annoy, makes bad experience for user from this page. So I've hide categories that we are in and advertisements that shows up on the sides.
I've also changed the descritpion of product to make it stick to let side and I've left all of possible advertisements under informations about items.
